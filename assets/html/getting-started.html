Core Theme Files

These are the nuts and bolts—the core assets—that comprise the FED of GenomeMag.com. We recommend that you not edit these files directly, but being aware of their contents will be a vital tool as you create new stylesheets and leverage the handy scripts included within.

  • /wp-content — This is where the Genome FED files are kept. There's a lot of 'building block' files here, but the really important ones are outlined below—these are the 'master' files pulled into the site (as you'll see later in the HTML markup).
    • /css —
      • genome-main.css — Genome's foundational styles—not a file we recommend you edit. It's also compressed, so it loads fast.
    • /js —
      • app.js — This file contains the code to initiate plugins other simple scripts.
      • sitescripts.js — Stores the primary plugins used throughout the site.
      • modernizr.js — Modernizr.js is a JavaScript library that detects HTML5 and CSS3 features in the user’s browser. This file is essential to presenting a beautiful, accessible site for all sorts browsers.
    • /min — Houses the minified versions of the above JS files
    • /fonts — Genome's Icon Font. Quickly and easily add icons on the fly: Genome Font Reference
    • /img Patterns, logos, icons, etc.

A Word on Working With the Code

The JS and CSS files referenced above are intended as a foundation—we recommend you don't edit these directly. When you inject your custom CSS styles into a page via the Wordpress backend, your code will be appended below the main CSS, genome-main.css: this gives your styles precedence so you can override the rules as you need to.
